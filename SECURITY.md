# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** create a public GitHub issue
2. Email: basel5001@users.noreply.github.com

## Important Security Notes

- Never commit tunnel credentials or tokens to git
- Rotate authentication tokens regularly
- Use strong, unique tokens for FRP/Rathole auth
- Enable HTTPS for all exposed HTTP services
- Restrict access to management dashboards (FRP dashboard, etc.)
- Monitor tunnel access logs for suspicious activity
